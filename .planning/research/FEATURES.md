# Ландшафт фич: трекер тренировок

**Домен:** веб-трекер силовых тренировок для новичков (local-first)
**Дата исследования:** 2026-04-14

## Table stakes (обязательно для v1)

Фичи, без которых продукт воспринимается как "слишком сырой" по меркам текущего рынка трекеров тренировок.

| Фича                                            | Почему ожидается                                                            | Сложность | Заметки                                                          |
| ----------------------------------------------- | --------------------------------------------------------------------------- | --------- | ---------------------------------------------------------------- |
| Быстрый лог подходов (вес/повторы в 1-2 тапа)   | Ключевое обещание категории — логировать "между подходами", без трения      | Medium    | Критично для вашей core value; должен работать без сети          |
| Автоподстановка прошлых значений по упражнению  | Пользователи ожидают быстрый прогрессивный перегруз без ручного вспоминания | Low       | Сильно ускоряет ввод и снижает ошибки                            |
| Шаблоны/рутины тренировок                       | Повторяемые сплиты (PPL/full-body/5x5) — базовый стандарт                   | Medium    | Для новичка: "запусти готовую тренировку и корректируй по месту" |
| Базовые графики прогресса (вес, повторы, объём) | Без визуализации пользователь не видит результат и теряет мотивацию         | Medium    | Достаточно трендов по неделям/месяцам, без сложной аналитики     |
| Категоризация упражнений                        | Нужна для быстрого выбора и фильтра в логе                                  | Low       | Начать с простых категорий: верх/низ/кор/тяга/жим и т.п.         |
| История тренировок + календарный просмотр       | Ожидаемый минимум для контроля регулярности                                 | Low       | Нужен быстрый переход "день -> тренировка -> упражнение"         |
| Экспорт данных (CSV/JSON)                       | В local-first сценарии пользователи ждут контроль над данными               | Medium    | Закрывает риск потери данных и повышает доверие                  |

## Differentiators (опциональное преимущество)

Фичи, которые повышают retention и ценность, но не обязательны для первичной поставки.

| Фича                                                                      | Ценность                                      | Сложность | Заметки                                          |
| ------------------------------------------------------------------------- | --------------------------------------------- | --------- | ------------------------------------------------ |
| Персональные инсайты для новичка ("что улучшить на следующей тренировке") | Превращает сырые логи в понятные действия     | High      | Делать rule-based, без ML в v1                   |
| Умные подсказки по прогрессии нагрузки                                    | Снижает тревожность "на сколько повышать вес" | Medium    | Простые правила на основе 2-3 последних сессий   |
| Тепловая карта нагрузки по группам мышц                                   | Наглядно показывает дисбаланс программы       | Medium    | Лучше как слой поверх существующей категоризации |
| Еженедельный digest "как прошла неделя"                                   | Повышает возвращаемость и ощущение прогресса  | Medium    | Можно строить на уже собранных агрегатах         |
| Готовые beginner-планы в 1 клик                                           | Снимает барьер старта для новой аудитории     | Medium    | Важно не перегружать количеством программ        |

## Anti-features (чего НЕ делать в v1)

| Anti-feature                                                               | Почему избегать                                                               | Что делать вместо                             |
| -------------------------------------------------------------------------- | ----------------------------------------------------------------------------- | --------------------------------------------- |
| Социальная лента, лайки, подписки                                          | Высокая стоимость разработки, низкая ценность для core сценария "быстрый лог" | Фокус на личном прогрессе и простом UX        |
| Cloud sync и аккаунты в первом релизе                                      | Резко усложняет архитектуру (конфликты, офлайн/онлайн синхронизация)          | Надёжный local-first + экспорт/импорт         |
| Перегруженная спортивная метрика (RPE/RIR/velocity/readiness) по умолчанию | Новичкам трудно, возрастает когнитивная нагрузка                              | Оставить только вес/повторы, расширения позже |
| Геймификация ради геймификации (бейджи/уровни без пользы)                  | Не решает главную задачу и размывает продукт                                  | Мотивация через реальный прогресс в графиках  |
| AI-генерация сложных программ на старте                                    | Риск недоверия и сложной валидации                                            | Простые проверенные шаблоны и правила         |

## Зависимости фич

```text
Категоризация упражнений -> Быстрый поиск упражнений при логе
Быстрый лог подходов -> История тренировок
История тренировок -> Графики прогресса
Автоподстановка прошлых значений -> Быстрый лог подходов
Шаблоны/рутины -> Быстрый лог подходов
Быстрый лог + Категоризация + История -> Недельный digest
Базовые графики прогресса -> Умные подсказки по прогрессии нагрузки
Категоризация + Графики -> Тепловая карта нагрузки по мышечным группам
```

## MVP-рекомендация (для вашего контекста)

Приоритет на ближайший milestone:

1. Быстрый лог (вес/повторы) + автоподстановка прошлых значений
2. Категоризация упражнений для быстрого выбора
3. Базовые графики прогресса (вес/повторы/объём)

Отложить:

- Социальные функции и коучинг — не усиливают core value для local-first новичка.
- Полноценный sync аккаунтов — нужен отдельный этап с проработкой конфликтов.
- Сложную аналитику (RPE-first/advanced readiness) — после подтверждения retention.
